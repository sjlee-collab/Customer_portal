문서 목적
Tableau, ArcGIS 기술지원, 계약/라이선스 문의, 고육 요청 등을 하나의 포탈에서 관리하고 내부 담당자가 Slack, Outlook, Salesforce 기반 업무 흐름에서 누락 없이 처리할 수 있도록 1차 개발 범위를 정의한다.

1. 한줄 요약
1차 고객지원 포탈은 고객 요청 접수와 이력 관리에 집중하는 최소 범위로 구축한다.
고객은 포탈에서 요청하고, 내부 담당자는 Slack/Outlook 알림으로 즉시 인지하며, 처리 상태와 답변은 포탈에 이력으로 남긴다.

2. 구축 배경
현재 고객지원 요청은 이메일, 전화, Slack, 영업 담당자 개별 연락 등 여러 채널로 분산되어 있다. 이로 인해 고객별 요청 이력 확인, 담당자 배정, 처리 상태 공유, 완료 여부 관리가 일관되게 이루어지기 어렵다.
또한 영업팀은 Salesforce를 사용하고, 내부 커뮤니케이션은 Slack과 Outlook/Office 365를 사용하고 있으므로 신규 시스템이 기존 업무 도구들을 대치하기보다 요청 접수와 이력의 기준 시스템 연결을 목표로 한다.

4. 1차 개발 기본 원칙
•	고객 화면은 단순하고 직관적으로 구성한다.
•	일반 조회 기능은 제공하지 않는다.
•	고객 지원 건은 내부 관리자가 생성한다.
•	고객은 소속 회사의 요청만 조회할 수 있다.
•	계약/라이선스 상세 정보는 포탈에 직접 표시하지 않는다.
•	Slack과 Outlook 통한 필수 알림은 자동화한다.
•	Salesforce를 기준 데이터로 고려하되, 최소 수동 등록으로 시작한다.
•	Notion은 부차적인 기록으로 두고, 1차 요청 처리는 포탈을 기준으로 한다.
•	GPT Business는 고객 직접 답변이 아니라 내부 업무 보조 기능으로 제한한다.

14. Salesforce 연동 방향
14.1 1차 권장 방안
1차에서는 수동 등록 시 Salesforce ID를 보관 방식으로 권장한다. 고객사 등록 시 이메일 도메인 또는 회사명을 기준으로 Salesforce Account ID, 계약 및 기술지원 현황, 담당자 참조 ID를 복제한다.

연동 방향	설명	우선순위
